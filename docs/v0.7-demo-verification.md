# v0.7 Demo Stack Verification

Tested: 2026-05-14

Host environment:

- Windows host, PowerShell shell.
- Docker 29.4.2.
- Docker Compose v5.1.3.
- Docker Desktop had to be started before the clean run because the engine was
  initially unavailable.

Pinned versions from `.env.demo.example`:

| Package | Version |
| --- | --- |
| `@isonia/types` | `0.7.0-alpha.2` |
| `@isonia/sdk` | `0.7.0-alpha.2` |
| `@isonia/theme-default` | `0.6.0-alpha.3` |
| `@isonia/control-plane` | `0.7.0-alpha.2` |
| `@isonia/evm-contracts` | `0.7.0-alpha.6` |
| `@isonia/app-core` | `0.7.0-alpha.4` |
| `isoniaos/docs` | `0.7.0-alpha.1` |

Commands run:

```sh
docker compose down -v --remove-orphans
rm -rf runtime
cp .env.demo.example .env
docker compose up --build
```

- `GET /v1/diagnostics`
- `GET /v1/diagnostics/indexer`
- `GET /v1/capabilities`
- `GET /v1/orgs`
- `GET /v1/orgs/1/finalization`

- `GET /v1/orgs/1/proposals/1/route`

Result:

- Clean build succeeded from the pinned tags.
- `contracts-deploy` completed and wrote runtime config plus seed output.
- Seeded orgs were org `1` and org `2`.
- Control Plane migrations completed.
- Control Plane API, indexer, and projection worker started.
- Initial indexing processed 51 events with projection backlog `0` and failed
  projections `0`.
- Capabilities reported serial activation and typed contract batch activation.
- Capabilities reported bootstrap organization finalization support.
- EIP-5792 wallet batching was reported unsupported and was not the default
  path.
- Both seeded orgs initially reported real `not_finalized` finalization state.
- App Core loaded `http://localhost:5173`, `/diagnostics`, org list, org
  overview, governance structure, proposal detail, and graph pages without a
  blank screen.
- Diagnostics displayed Control Plane connection, chain ID `31337`, GovCore and
  GovProposals addresses, typed batch activation support, serial fallback,
  disabled EIP-5792, and finalization status.

Write-flow checks:

- The in-app browser used for verification did not expose an injected wallet, so
  the App Core wallet transaction modal flow could not be completed in this run.
- A local Hardhat finalization transaction was sent from the seeded org `1`
  bootstrap admin to verify the contract, event, indexing, and read-model path.
- After finalization, Control Plane indexed block `56`, processed 52 total raw
  events, projected one `OrganizationFinalized` event, and kept backlog and
  failed projections at `0`.
- `GET /v1/orgs/1/finalization` updated to `finalized` with
  `bootstrapAdminMutationsAllowed: false` and the expected blocked bootstrap
  admin operations.
- Direct post-finalization static calls to serial `createBody` and typed
  `batchCreateBodies` reverted.
- App Core then showed org `1` as `FINALIZED`; the setup surface showed
  bootstrap admin changes closed and the concrete bootstrap action button was
  disabled.

Known non-blocking warnings:

- App Core build reported Vite chunk-size warnings for large web3 bundles.
- Browser console logs repeated warnings while probing optional
  `/isonia.config.local.json`; runtime config still loaded from
  `/isonia.config.json`.
- README and `.env.demo.example` still list `DOCS_VERSION=0.7.0-alpha.1`. This
  is not a runtime failure because the stack does not clone docs at runtime, but
  it should be handled in a separate bookkeeping update if docs alignment should
  point at `v0.7.0-alpha.2`.
- The App Core home page still displays `v0.6 local demo` copy even though the
  pinned app package version is `0.7.0-alpha.4`.

Final verdict:

- Clean build, deploy, seed, diagnostics, capabilities, read models, finalization
  indexing, and post-finalization restriction behavior passed.
- App Core wallet-driven activation/finalization could not be fully verified in
  this environment because no injected wallet provider was available.
